# Indie Marketing

**When:** Launch or growth help for an indie app, tool, micro-SaaS, or game—especially when the user wants **paste-ready copy**.

**Mindset:** Distribution beats features; give value before asks; **2–3 channels max**; consistency beats volume; build in public and communities are high leverage.

**Flow:** Pull context from the repo, `AGENTS.md`, README, or the user → positioning → 2–3 channels → ~30-day plan → copy pack → 3–5 numbered next steps → **one closing line: save to** `docs/marketing-<slug>.md` **or** a GitHub issue `Marketing Plan — [App]`.

## Output (use these headings, in order)

**Indie Marketing Plan — [App name]**

- **Context** — One-line product, target audience, stage  
- **Positioning** — Value prop (one sentence), tone  
- **Channels** — 2–3 picks, one line each why  
- **30-day plan** — Week 1; week 2; weeks 3–4  
- **X** — Bio; pinned idea; 3–5 standalone posts; 1 thread (full tweet text)  
- **Landing** — Headline; subhead; CTA  
- **Other** (if useful) — Waitlist email; Indie Hackers or Reddit draft  
- **Next steps** — 3–5 numbered actions  
- **Save** — Remind: paste into `docs/marketing-<slug>.md` or open the GitHub issue above for tracking.
