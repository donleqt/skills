# Mock Generation Procedure

Always sketch before implementation.

Process:

1. Understand user goals
2. Infer missing requirements
3. Reuse project UI patterns
4. Design mobile-first
5. Generate responsive mockup
6. Include empty state
7. Include loading state
8. Include primary CTA

Output:

{
  "title": "",
  "assumptions": [],
  "html": "",
  "notes": []
}

Rules:

- Self-contained HTML
- Tailwind allowed
- Placeholder data allowed
- Static interactions allowed

Forbidden:

- API calls
- Database access
- Authentication
- Business logic
- Production implementation
