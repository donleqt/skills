# Responsive Rules

Always design mobile-first.

Support:

- Mobile
- Tablet
- Desktop

Patterns:

Mobile:
- Single column
- Stacked layout

Tablet:
- Two-column adaptation

Desktop:
- Multi-column grid

Avoid:

- Fixed widths
- Desktop-first layouts
- Pixel-perfect assumptions

Validate (see `references/validate-mock.md` for full checklist):

- Page padding and section spacing at all breakpoints
- No horizontal scroll on mobile
- Readability
- Accessibility
- CTA visibility
- Navigation discoverability
- Browser verification at 375px, 768px, 1280px before delivery
