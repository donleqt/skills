# Idea validation

**When:** User wants an honest decision before building (app, tool, micro-SaaS, AI wrapper, game).

**Triggers:** validate my idea, is this worth building, should I build this, time to first revenue, will this make money, idea assessment.

**Benchmarks (use in reasoning, don’t lecture):**
- ~45–54% of indie SaaS never earns; median MRR is tiny (~$25–$100); top ~30% take most revenue.
- First pay: days if validated + audience; 6+ months if distribution is weak.
- ~$1K MRR: often 6–12 months when it happens; many never get there.
- Strong signals: **8–12%** landing signup rate **or** **5–10** pre-sale commitments.
- AI speeds build; **willingness to pay** still wins.

## Output (always this shape)

**Idea Validation: [name]**

**Summary** — Audience · Pain · Solution (one line each or bullets).

**7–14 day plan**
1. **Landing smoke** — Page + headline + pricing + waitlist; 200–500 targeted visits (X, Reddit, IH, niche). Pass: **≥8–12%** signup.
2. **Pre-sale / fake door** — Discounted lifetime or paid beta **$19–$49**. Pass: **5–10** paid or equivalent intent.
3. **Interviews** — **8–12** people; past behavior (“how do you handle this today?”) and real budget.

**Time to first earning** — Optimistic **4–8 wk** · Realistic **3–6 mo** · Pessimistic **6–12+ mo** (from today).

**If launch + execution go well**
- Months 1–3: users **50–500**, MRR **$0–$300**
- Year 1 realistic MRR: **$200–$2,000**; optimistic: **$3,000–$10,000+**
- **$1K MRR in 12 months:** Low / Medium / High (many micro-SaaS stay **under $1K** lifetime)

**Recommendation → [DO IT / DON'T DO IT / PROCEED WITH CAUTION]**

**Reasoning** — 2–4 bullets: signals & red flags · saturation · distribution · monetization.

**Next 7 days** — 3–4 numbered actions (specific to their idea).

**Save** — User pastes full output to `docs/idea-validation/[slug].md` or a GitHub issue titled `Idea Validation - [name]`.

Offer one concrete help line only if useful (e.g. headline or interview opener).
